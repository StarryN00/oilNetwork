# 企油通油站伙伴平台 V1 产品规划

更新时间：2026-06-25

## 一、系统定位

企油通要建立油站网络，核心不是给油站“卖系统”，而是帮助油站获得可量化、可运营、可复购的物流企业增量生意。

因此，V1 的系统定位建议为：

> **企油通油站伙伴平台：面向民营油站的增量生意运营系统。**

它不是传统油站管理系统，也不是复杂的油站 ERP，而是围绕企油通导流交易、优惠活动、客户归因和对账看板，帮助油站看见增量、控制让利、沉淀企业客户。

---

## 二、为什么先从民营油站开始

两桶油体系中，第三方系统接入和铺设难度较高，涉及总部、省公司、油站系统、支付系统、营销体系等多层决策，推进周期长。

民营油站则更关注实际经营结果，尤其是：

- 能不能带来新增车流；
- 能不能带来稳定的大车/物流车加油量；
- 优惠是否可控；
- 能不能减少对账和开票麻烦；
- 是否不需要大规模系统改造。

目前已有同事尝试与民营油站沟通，若以“带增量”的方式切入，油站接受度较高。因此 V1 应优先服务民营油站，先用数据证明价值，再逐步沉淀油站网络。

---

## 三、油站真实诉求判断

油站不会因为“企油通需要油站网络”而接入，油站只会因为系统能帮助它解决经营问题而接入。

油站真实关心的不是系统本身，而是以下几件事：

### 1. 是否能带来稳定车流和销量

油站最直接的问题是：

- 今天有没有车来；
- 来的是不是大车、物流车；
- 是否形成稳定月度加油量；
- 这些客户是不是企油通带来的。

企油通应帮助油站把分散的物流企业和司机流量组织起来，形成可预期的企业客户池。

### 2. 优惠是否可控

油站愿意让利，但不愿意无差别降价。

油站担心：

- 优惠给了散客，没有带来增量；
- 优惠被薅羊毛；
- 无法判断是不是目标客户；
- 让利成本不可控。

企油通需要通过企业、司机、车辆、Token、活动规则等方式，让油站实现精准优惠。

### 3. 是否需要改造油站系统

民营油站普遍不愿意做重改造。如果要求改 POS、ERP、收银系统、开票系统，推进会明显变慢。

V1 应坚持轻改造或零改造原则：

- 先用交易数据导入/接口同步；
- 通过 Token 标识客户和交易归属；
- 通过活动规则实现优惠管理；
- 通过支付机构/银联营销能力实现消费减免；
- 不要求油站先改造核心收银系统。

### 4. 是否能沉淀企业客户资产

如果油站只是给优惠，交易结束后客户仍然没有沉淀，油站动力不足。

企油通应帮助油站形成企业客户运营能力：

- 哪些物流企业贡献大；
- 哪些司机/车辆高频复购；
- 哪些客户正在流失；
- 哪些企业值得给专属优惠；
- 哪些时间段适合做定向活动。

---

## 四、V1 核心目标

V1 不追求完整油站 SaaS，而是验证一句话：

> **企油通是否真的能给民营油站带来可量化的增量交易，并让油站愿意继续配合优惠。**

因此，V1 只做四件事：

1. **油站看得见增量**  
   展示企油通带来的交易额、司机数、车辆数、复购率、优惠成本、预计增量毛利。

2. **油站配得了活动**  
   支持油站配置简单、可控的优惠活动。

3. **企油通识别得了客户/司机/车辆**  
   通过 Token、企业、司机、车辆等字段识别交易归属和优惠资格。

4. **双方对得清账**  
   支持订单、支付、优惠、发票、活动效果的明细查询和导出。

---

## 五、V1 产品定义

建议内部定义为：

> **油站伙伴平台 V1：围绕民营油站，提供企油通导流交易、优惠活动、客户归因和对账看板，帮助油站看见增量、控制让利、沉淀企业客户。**

系统不应以“功能复杂”为目标，而应以“让油站愿意继续参与企油通网络”为目标。

---

## 六、核心业务口径

### 1. 增量交易口径

V1 必须定义“什么叫企油通带来的增量”，否则油站可能质疑：这些车本来就会来，不应算企油通贡献。

V1 可先采用简单口径：

- 首次通过企油通在该油站交易的司机/车辆；
- 指定活动期间通过 Token 识别完成的交易；
- 由企油通企业客户带来的交易；
- 历史未在该站交易、但通过企油通活动触达后到站交易的司机/车辆。

后续可升级为更复杂的归因模型。

### 2. 优惠活动口径

V1 的活动规则应轻量，先支持：

- 每升优惠；
- 满减；
- 固定金额立减；
- 指定企业；
- 指定司机；
- 指定车辆；
- 指定油站；
- 指定活动时间；
- 总预算；
- 单车/单司机限额。

不要一开始做复杂营销规则。

### 3. Token 口径

Token 是底层能力，不是附属功能。

Token 至少承担三个作用：

1. 标识客户是谁；
2. 标识这笔交易是否属于企油通；
3. 标识优惠、补贴、数据归因。

Token 应贯穿企业、司机、车辆、活动、订单、对账等核心表。

### 4. 对账口径

V1 至少要支持：

- 订单明细导出；
- 活动优惠明细导出；
- 企业客户交易汇总；
- 月度对账单；
- 支付金额、优惠金额、实付金额、发票状态等字段。

哪怕初期只是 CSV/Excel，也必须支持导出。

## 七、V1 核心页面设计

### 1. 油站驾驶舱

目标：让油站一眼看到企油通带来的增量生意。

核心指标：

- 今日交易额；
- 本月交易额；
- 今日交易笔数；
- 本月交易笔数；
- 加油升数；
- 客单价；
- 新增司机数；
- 复购司机数；
- 复购率；
- 已发放优惠；
- 预计增量毛利。

建议页面标题：

> 今天企油通为你带来的增量生意

### 2. 活动管理

目标：让油站可以配置简单、可控的优惠活动。

核心字段：

- 活动名称；
- 活动油站；
- 活动时间；
- 活动油品；
- 优惠方式：每升优惠 / 满减 / 固定金额立减；
- 适用客户：全部企油通客户 / 指定企业 / 指定司机 / 指定车辆；
- 总预算；
- 单车限额；
- 单司机限额；
- 每日限额；
- 活动状态：未开始 / 进行中 / 已结束 / 已暂停；
- 活动效果：交易金额、交易笔数、优惠金额、参与司机数、复购情况。

### 3. 企业客户看板

目标：帮助油站沉淀企业客户资产。

核心字段：

- 企业名称；
- 司机数；
- 车辆数；
- 加油金额；
- 加油升数；
- 优惠金额；
- 交易笔数；
- 复购率；
- 最近加油时间；
- 是否适合专属优惠；
- 客户贡献排名。

未来可扩展为油站企业客户 CRM。

### 4. 司机/车辆复购分析

目标：帮助油站判断客户是否稳定。

核心字段：

- 首次加油司机；
- 二次复购司机；
- 高频司机；
- 流失司机；
- 车辆号；
- 所属企业；
- 常用油品；
- 常来时间段；
- 最近一次交易时间；
- 累计交易额；
- 累计优惠金额。

### 5. 订单与对账

目标：让油站和企油通对得清账。

核心字段：

- 订单号；
- 交易时间；
- 油站；
- 企业；
- 司机；
- 车辆；
- Token；
- 油品；
- 升数；
- 原价金额；
- 优惠金额；
- 实付金额；
- 支付方式；
- 支付状态；
- 发票状态；
- 活动名称；
- 是否增量交易；
- 异常状态。

基础功能：

- 条件筛选；
- 明细查询；
- Excel/CSV 导出；
- 月度对账单导出。

### 6. Token / 客户识别

目标：建立企油通交易归因和优惠控制底座。

核心字段：

- Token 编号；
- 归属企业；
- 司机；
- 车辆；
- 绑定手机号；
- 适用油站；
- 适用活动；
- 生效时间；
- 失效时间；
- 状态：有效 / 暂停 / 失效；
- 已使用次数；
- 已使用金额；
- 已享优惠金额。

## 九、V1 开发策略

### 1. 先做 Web 管理后台原型

建议优先开发一个可演示、可迭代的 Web 管理后台，覆盖：

1. 油站驾驶舱；
2. 活动管理；
3. 企业客户看板；
4. 订单与对账；
5. Token/客户识别。

### 2. 先用模拟数据/导入数据跑通

不要一开始卡在支付机构、银联、油站 POS 对接。

V1 可先支持：

- 手工导入交易流水；
- 模拟 Token 识别；
- 模拟活动优惠；
- 模拟增量归因；
- 后续再接真实接口。

### 3. 先验证价值，再深化集成

第一阶段目标不是系统完整，而是让油站看到：

- 企油通带来了多少交易；
- 优惠花了多少钱；
- 哪些企业和司机值得继续服务；
- 月度账能不能对清楚。

只要这件事成立，后续再扩展支付接口、银联营销、电子发票、油站系统对接。

## 十、后续可扩展方向

V1 跑通后，可逐步扩展：

1. **运营建议**
   - 哪些客户快流失；
   - 哪些司机可召回；
   - 哪些时段适合做活动；
   - 是否建议加大优惠。

2. **油站企业客户 CRM**
   - 企业客户分层；
   - 专属优惠策略；
   - 客户经理跟进；
   - 企业客户生命周期。

3. **银联营销能力接入**
   - 消费减免；
   - 指定油站活动；
   - 指定客户 Token；
   - 活动预算控制。

4. **发票与证据包联动**
   - 油票；
   - 小票；
   - ETC；
   - 支付订单；
   - 运单；
   - GPS；
   - 单票证据材料包。

5. **金融产品联动**
   - 企业授信；
   - 运单融资；
   - 应收账款融资；
   - 司机货到即付；
   - 银行风控数据输出。

## 十一、阶段性结论

企油通油站网络的建设，不能从“铺油站系统”出发，而应从“帮助油站获得可量化增量”出发。

V1 的关键不是功能多，而是能否让民营油站清楚看到：

- 企油通带来了多少新增交易；
- 优惠有没有换来真实销量；
- 哪些企业和司机值得持续运营；
- 钱和票是否能对清楚。

因此，V1 应围绕四个核心能力建设：

1. **增量可视化**
2. **优惠活动配置**
3. **Token客户识别**
4. **订单对账导出**

当这四项能力跑通后，企油通才具备继续扩大民营油站网络、向两桶油区域样板突破、以及进一步叠加金融产品的基础。
